## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, copy button margin, inline padding (`--space-xs`) |
| sm | 8px | Compact element spacing, button gap, step row padding (`--space-sm`) |
| md | 16px | Default element spacing, button horizontal padding, section padding (`--space-md`) |
| lg | 24px | Section padding, quick actions block vertical padding (`--space-lg`) |
| xl | 32px | Layout gaps (`--space-xl`) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Quick action buttons: `min-height: 44px` for accessible touch targets (matches StepChecklist row pattern)
- "Try another way" button: `min-height: 32px` — secondary inline action, does not need full touch-target height since it sits within a scrollable content area on desktop

## Color

Source: `src/styles/theme.css` — both light and dark mode tokens carried forward.

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| Dominant (60%) | `rgba(255,255,255,0.72)` | `rgba(28,28,30,0.92)` | Overlay surface background |
| Secondary (30%) | `rgba(0,0,0,0.05)` | `rgba(255,255,255,0.08)` | Quick action button resting background, "Try another way" resting background |
| Accent (10%) | `#007AFF` | `#0A84FF` | Reserved for: "Ask" button border/ring only (distinguishes it as the freeform escape hatch), loading spinner on quick action tap |
| Destructive | `#FF3B30` | `#FF453A` | Not used in this phase |

Accent reserved for: "Ask" button ring/border treatment (signals different behavior from Fix/Explain/Optimize), and the brief loading indicator on any button that is mid-flight.

Color rules specific to this phase:
- Quick action buttons (Fix, Explain, Optimize) at rest: background `--color-surface-secondary`, border `--color-border` (1px), text `--color-text-primary`.
- Quick action buttons on hover: background tints to `--color-surface-secondary` at 1.5x opacity via filter or explicit rgba — no accent color on hover for Fix/Explain/Optimize.
- "Ask" button at rest: background `--color-surface-secondary`, border `1px solid var(--color-accent)` at 40% opacity, text `--color-text-primary`. Distinguishes it as a different action type without full accent fill.
- Button in loading state (screenshot being taken + AI call firing): opacity 0.5, pointer-events: none. No spinner icon required — opacity degradation is sufficient for an overlay that returns quickly.
- "Try another way" at rest: no background, no border. Text `--color-text-secondary`. On hover: text `--color-text-primary`. Inline text-button treatment.

## Component Inventory

New components introduced in Phase 11:

### `QuickActions.tsx`

Renders in the `Show when={lastIntent().length === 0}` branch of SidebarShell.

Layout: 2×2 grid. Four equal-width buttons. Grid gap: `--space-sm` (8px). Container padding: `--space-md` (16px) vertical, `--space-md` horizontal. Full-width within sidebar (sidebar width: 300px).

Button anatomy:
- `min-height: 44px`
- `border-radius: var(--radius-md)` (10px)
- `padding: var(--space-sm) var(--space-md)` (8px 16px)
- `border: 1px solid var(--color-border)`
- `background: var(--color-surface-secondary)`
- `font-size: var(--font-size-body)` (14px)
- `font-weight: var(--font-weight-regular)` (400)
- `color: var(--color-text-primary)`
- `cursor: pointer`
- `transition: var(--transition-fast)` (150ms ease)
- "Ask" button: `border: 1px solid color-mix(in srgb, var(--color-accent) 40%, transparent)` — fallback to `rgba(0,122,255,0.40)` / `rgba(10,132,255,0.40)` in dark.

Disabled state (while loading): `opacity: 0.5`, `pointer-events: none`, `cursor: not-allowed`.

Props interface:
```
interface QuickActionsProps {
  onAction: (preset: string) => void;  // called with preset prompt string for Fix/Explain/Optimize
  onAsk: () => void;                   // focuses text input — does not call onAction
  disabled?: boolean;                  // true while an AI call is in-flight
}
```

Button labels and presets (from CONTEXT.md D-03):
- "Fix" → `"Fix the issue shown in the screenshot"`
- "Explain" → `"Explain what's happening in the screenshot"`
- "Optimize" → `"How can I improve or optimize what's shown"`
- "Ask" → calls `onAsk()`, focuses text input, does not fire AI call

ARIA: each button has `aria-label` equal to its label text. Container has `role="group"` and `aria-label="Quick actions"`.

### `TryAnotherWay.tsx`

Renders below the active StepChecklist / RawGuidanceText area when `contentState() === "done"`.

Layout: single inline text-button, left-aligned, padding `--space-sm` vertical, `--space-md` horizontal.

Anatomy:
- No background, no border
- `font-size: var(--font-size-label)` (12px)
- `font-weight: var(--font-weight-regular)` (400)
- `color: var(--color-text-secondary)` at rest
- `color: var(--color-text-primary)` on hover
- `transition: var(--transition-fast)` (150ms ease)
- `min-height: 32px`
- `cursor: pointer`
- `display: inline-flex`, `align-items: center`

Disabled state: `opacity: 0.4`, `pointer-events: none` while loading.

Props interface:
```
interface TryAnotherWayProps {
  onRetry: () => void;  // triggers re-run with "different approach" prompt modifier
  disabled?: boolean;
}
```

ARIA: `aria-label="Try another way"`.

## Interaction Contract

### Empty state (ACTN-01)

Trigger: `lastIntent().length === 0`

Rendered: QuickActions 2×2 grid. Text input field present below the button grid, de-emphasized (not hidden — "Ask" focuses it). No empty-state illustration or heading — buttons ARE the content.

Layout stacking order from top to bottom in empty state:
1. Task header strip (if `taskTitle()` is set from prior session — persists across hide/show per Phase 9 D-11)
2. QuickActions grid (primary visual weight)
3. Text input (below, same size as normal, but no cursor focus on overlay open — focus goes to none, letting the buttons draw the eye)
4. Region thumbnail strip (if `thumbnailB64()` is set)

### Region-select + quick actions (ACTN-02)

Trigger: `onRegionSelected` fires, `thumbnailB64()` is set, `lastIntent().length === 0`

Rendered: Same QuickActions grid. The thumbnail strip sits above the buttons as visual confirmation. Tapping Fix/Explain/Optimize calls `submitIntent()` with the region screenshot captured at tap time (matches existing `submitIntent` flow which already uses `selectedRegion()`).

No new button set. Same 4 buttons, same 2×2 grid. No AI-suggested buttons in this phase (ACTN-04 deferred per CONTEXT.md D-08).

### Button tap → loading state (ACTN-04 partial)

Timing: button tap triggers `submitIntent()` synchronously. `contentState` transitions to `"loading"` immediately. QuickActions component receives `disabled={true}` → all 4 buttons render at opacity 0.5, non-interactive. Target: button → loading transition visible within 100ms (no async work before state update).

### "Try another way" (ACTN-03)

Trigger: `contentState() === "done"`

Placement: immediately below the StepChecklist or RawGuidanceText block, left-aligned, above the text input.

Behavior: calls a handler in SidebarShell that re-runs `submitIntent()` with the last intent plus a prompt modifier: `" — suggest a meaningfully different approach than before"`. New response replaces the current exchange (same pattern as existing `handleRetry()`). "Try another way" button disappears during loading and reappears on the next `"done"`.

Does not stack responses. New guidance replaces the current exchange.

### "Ask" button

Tap: focuses `inputRef` (the existing TextInput). Does not fire AI. Does not close or hide the QuickActions grid — grid disappears naturally when `lastIntent().length > 0` (after user types and submits).

### Error state (screenshot capture failure)

If `captureScreenshot()` throws on a quick action tap: `screenshotFailed` signal is set to true (existing pattern in SidebarShell). The existing error rendering path handles this. Quick action buttons return to enabled state. No new error UI required for this phase.

| Element | Copy |
|---------|------|

| Primary CTA (Explain) | Explain |

| Primary CTA (Ask) | Ask |

| Empty state body | (none — buttons communicate function without supporting text) |

Copywriting rules:
- Button labels are single words, sentence case. No icons on buttons.
- "Try another way" is lowercase after "Try" — matches secondary inline text convention.
- No exclamation marks. No marketing copy. Labels name the action, not the outcome.

## Accessibility Contract

- All quick action buttons: `min-height: 44px` (WCAG 2.5.5 touch target)
- "Try another way": `min-height: 32px` — acceptable for a secondary desktop action in a narrow sidebar
- Disabled buttons: `aria-disabled="true"` + `pointer-events: none` (opacity alone is not sufficient for accessibility)
- Loading state: `aria-busy="true"` on the QuickActions container while `disabled` is true
- Color contrast: all text uses `--color-text-primary` (light: `rgba(0,0,0,0.88)`, dark: `rgba(255,255,255,0.92)`) — passes WCAG AA at 14px
- Keyboard: all buttons reachable via Tab. "Ask" button focuses the text input on Enter/Space.

## Implementation Notes

- Use SolidJS inline `style` prop throughout (matches established pattern in StepChecklist.tsx and SidebarShell.tsx — no className/CSS modules).
- All color values via `var(--color-*)`. No hardcoded hex values in component files.
- All spacing via `var(--space-*)`. No hardcoded px in component files.
- `lucide-solid` is the only icon library. Phase 11 components use no icons (text-only buttons).
- QuickActions slots into the existing `Show when={lastIntent().length === 0}` branch in SidebarShell — replaces or wraps the `EmptyState` component currently there.
- TryAnotherWay slots below `currentExchange()` rendering, above `TextInput`, conditional on `contentState() === "done"`.
