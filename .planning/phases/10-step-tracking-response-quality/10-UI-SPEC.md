## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, copy button gap to text |
| sm | 8px | Between checklist rows, between step icon and label |
| md | 16px | Content area horizontal padding |
| lg | 24px | Section breaks |
| xl | 32px | Layout gaps (not used in this phase) |
| 2xl | 48px | Major section breaks (not used in this phase) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Step row: `min-height: 44px` ‚Äî accessibility touch target, not a spacing unit. Not derived from the 8-point scale; required by accessibility contract (CONTEXT.md D-10).
- Step left border (current step highlight): `3px solid var(--color-accent)` ‚Äî decorative, not spacing.
- Step left border (non-current): `3px solid transparent` ‚Äî holds column width so text does not shift on toggle.

Source: existing scale from `theme.css`. Exceptions derived from CONTEXT.md D-10 and D-03.

### SessionFeed (MODIFIED ‚Äî `src/components/SessionFeed.tsx`)

Phase 10 change: no longer receives `currentExchange` as part of `sessionHistory`. The `sessionHistory` prop contains prior turns only. Active exchange is rendered by `StepChecklist` or `RawGuidanceText` below SessionFeed.

`streamingText` prop receives an empty string when `contentState !== "streaming"`.

Visual treatment of prior session history: unchanged from Phase 9 (flat muted text, `var(--color-text-secondary)`). No checklist, no copy buttons on historical items.

## Interaction Contracts

### Step Toggle (STEP-02)

| Trigger | Effect |
|---------|--------|
| Click any step row | Flip `completed` for that index: `setSteps(prev => prev.map((s, i) => i === idx ? { ...s, completed: !s.completed } : s))` |
| After any toggle | "Current step" highlight auto-advances to first uncompleted item (derived, not stored separately) |
| All steps completed | No "current" highlight ‚Äî all rows render as completed. No automatic action taken. |

No enforced linear order. Non-linear clicks are fully supported (D-04).

### Step Parse (D-01, D-02)

| Trigger | Action |
|---------|--------|
| `onDone` fires | Call `parseSteps(accumulatedText)` |
| Text starts with `"1."` | Parse all `/^\d+\.\s+(.+)$/` matches ‚Üí produce `Step[]` ‚Üí `setSteps(result)` |
| Text does NOT start with `"1."` | `parseSteps` returns `[]` ‚Üí `setSteps([])` ‚Üí RawGuidanceText shown |
| After parse | `setCurrentExchange({ intent: lastIntent(), guidance: accumulatedText })` |

Parser skips non-matching lines (code blocks, blank lines). If result array is empty after parsing, `[]` is returned (triggers fallback).

### Copy Button (D-08, D-09)

| Trigger | Action |
|---------|--------|
| Click copy button | Call `copyToClipboard(text)` |
| `navigator.clipboard.writeText` succeeds | Silent success ‚Äî no toast, no icon change |
| `navigator.clipboard.writeText` fails | Fall back to `execCommand("copy")` via hidden textarea |
| Both paths fail | Silent ‚Äî no error shown to user |

No confirmation feedback in Phase 10. Deferred: success checkmark state.

### Session Exchange Lifecycle

| Moment | `currentExchange` | `sessionHistory` | `steps` |
|--------|-------------------|------------------|---------|
| Overlay open (idle) | `null` | `[]` (or retained from prior session) | `[]` |
| `submitIntent` called | Move to history if non-null, then set to `null` | Appended (cap 3) | Reset to `[]` |
| Streaming | `null` | unchanged | `[]` |
| `onDone` | Set from `accumulatedText` | unchanged | Set from `parseSteps()` |
| `handleNewTask` | `null` | `[]` | `[]` |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Step checklist container | `aria-label="Step checklist"` + `aria-live="polite"` |
| Each step row button | `aria-label="Step N: [label]"` |
| Copy button | `aria-label="Copy code block"` or `"Copy command"` |
| Min touch target | `min-height: 44px; min-width: 44px` on all interactive elements (existing pattern, D-10) |
| Step icon alt | Icons are decorative ‚Äî no `aria-label` on icon element itself; label is on the parent `<button>` |
| Completed step | `text-decoration: line-through` on label ‚Äî do not remove from DOM (screen reader still reads it) |
| Color contrast | Step labels at `var(--color-text-primary)` (0.88 opacity) meet WCAG AA. Completed labels at `var(--color-text-secondary)` (0.50 opacity) are marginal ‚Äî acceptable for decorative completed state, not for actionable elements. |
