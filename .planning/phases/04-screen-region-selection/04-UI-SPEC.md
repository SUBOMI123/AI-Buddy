## Interaction States and Flow

### State Machine Addition to `ContentState`

The existing type in `SidebarShell.tsx`:
```
type ContentState = "empty" | "loading" | "streaming" | "error" | "listening"
```

Extends to:
```
type ContentState = "empty" | "loading" | "streaming" | "error" | "listening" | "selecting"
```

`"selecting"` is the transient state while the `region-select` Tauri window is open. The sidebar is hidden during this state.

A separate signal `selectedRegion: Accessor<RegionCoords | null>` tracks whether a region is locked in (drives thumbnail visibility). This is NOT part of `ContentState`.

### Interaction Sequence

| Step | User Action | UI Response |
|------|------------|-------------|
| 1 | User clicks Crop button in sidebar | Sidebar hides. `region-select` window opens covering full primary monitor. `contentState` set to `"selecting"`. |
| 2 | User moves cursor over overlay | Cursor is `crosshair`. Instruction text visible. Dark scrim covers screen. |
| 3 | User clicks and drags | Rubber-band rectangle draws with accent border + tint fill. Instruction text hides during drag. |
| 4 | User releases mouse (region >= 10px) | `region-selected` Tauri event emitted with physical pixel coords. Overlay window hides. Sidebar reappears. `contentState` returns to `"empty"`. `selectedRegion` set to coords. Thumbnail preview appears above input. |
| 5 | User releases mouse (region < 10px) | Treated as accidental click. `region-cancelled` event emitted. Overlay hides. Sidebar reappears. No region set. `contentState` returns to `"empty"`. |
| 6 | User presses Escape during draw | `region-cancelled` event emitted. Overlay hides. Sidebar reappears. No region set. `contentState` returns to `"empty"`. |
| 7 | User clicks X on thumbnail | `selectedRegion` set to `null`. Thumbnail removed. Input area returns to normal. Fallback to full-screen capture on next submit. |
| 8 | User submits query with region set | `captureRegion(coords)` called instead of `captureScreenshot()`. After submit completes, `selectedRegion` resets to `null` (D-07 — region clears per submit). |
| 9 | User submits query without region | `captureScreenshot()` called (Phase 2 fallback behavior). No change. |

## Constraints and Invariants

The following constraints are absolute and must not be violated by the executor:

1. **Coordinate invariant:** Physical pixel coordinates (not logical/CSS pixels) MUST be passed to `capture_region`. Apply `scaleFactor()` multiplication before emitting `region-selected` event. (D-09, RESEARCH.md Pattern 3)

2. **Overlay z-order invariant:** The `region-select` Tauri window MUST have `alwaysOnTop: true` and `set_focus()` called after `show()`. If the overlay appears behind target app windows, the core mechanic breaks. (RESEARCH.md Pitfall 2)

3. **Overlay sizing invariant:** Declare `fullscreen: true` in `tauri.conf.json` OR set `PhysicalSize` from `primary_monitor()` at show time. Never use fixed pixel dimensions. (RESEARCH.md Pitfall 3)

4. **Tab focus invariant:** The root div of `RegionSelect.tsx` MUST have `tabIndex={0}` and `.focus()` called on mount. Without this, Escape key does not cancel selection. (RESEARCH.md Pitfall 4)

5. **Region reset invariant:** `selectedRegion` MUST be set to `null` after every successful submit. Each query starts with no region. (D-07)

6. **Sidebar visibility invariant:** Sidebar MUST reappear after every overlay close — whether by successful selection, Escape, or accidental tiny drag. User must never be left with neither sidebar nor overlay visible. (D-10)

7. **Minimum region size invariant:** Region smaller than 10×10 logical pixels on mouseup MUST be treated as cancel, not selection. (RESEARCH.md Pitfall 6)
