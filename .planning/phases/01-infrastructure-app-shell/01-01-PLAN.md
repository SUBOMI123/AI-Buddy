Create the `worker/` directory at the project root. Initialize a new npm project inside it.

1. Create `worker/package.json` with:
   - name: "ai-buddy-proxy"
   - dependencies: `hono` (^4.12.0)
   - devDependencies: `wrangler` (^4.81.0), `@cloudflare/workers-types` (latest)
   - scripts: `"dev": "wrangler dev"`, `"deploy": "wrangler deploy"`, `"test": "npx tsx --test src/index.test.ts"`

2. Create `worker/tsconfig.json` with:
   - target: "ESNext", module: "ESNext", moduleResolution: "Bundler"
   - types: ["@cloudflare/workers-types"]
   - strict: true

3. Create `worker/wrangler.toml` with:
   - name = "ai-buddy-proxy"
   - main = "src/index.ts"
   - compatibility_date = "2024-09-23"
   - No secrets in this file (API keys set via `npx wrangler secret put`)
   - **KV namespace binding (per D-10):**
     ```toml
     [[kv_namespaces]]
     binding = "RATE_LIMIT"
     id = "placeholder-create-via-wrangler"
     ```
     Note: The actual KV namespace ID is created via `npx wrangler kv namespace create RATE_LIMIT` before deployment. The placeholder ID is fine for local dev (`wrangler dev` creates a local KV automatically).

4. Create `worker/src/index.ts` with a Hono app (per D-09):

   Define TypeScript `Bindings` type:
   ```
   type Bindings = {
     ANTHROPIC_API_KEY: string;
     ASSEMBLYAI_API_KEY: string;
     ELEVENLABS_API_KEY: string;
     RATE_LIMIT: KVNamespace;
   };
   ```

   **Rate limiting helper (per D-10):** Create an async function `checkRateLimit(kv: KVNamespace, token: string): Promise<{ allowed: boolean; remaining: number }>` that:
   - Uses KV key format: `rate:{token}` with a 60-second TTL
   - Reads the current count from KV. If null, starts at 0.
   - If count >= 60, returns `{ allowed: false, remaining: 0 }`.
   - Otherwise, increments count and writes back with `expirationTtl: 60`.
   - Returns `{ allowed: true, remaining: 60 - newCount }`.

   **Auth middleware (per D-10):** Create a Hono middleware that:
   1. Reads the `x-app-token` header.
   2. Validates the token is a non-empty string of at least 32 characters (UUID format).
   3. If missing or invalid, returns 401 `{ error: "Unauthorized" }`.
   4. Calls `checkRateLimit(c.env.RATE_LIMIT, token)`.
   5. If rate limited, returns 429 `{ error: "Rate limit exceeded", retry_after: 60 }` with `Retry-After: 60` header.
   6. Sets `X-RateLimit-Remaining` header on the response with the remaining count.
   Apply this middleware to all routes except `/health`.

   **Routes** (per D-11):

   - `GET /health` -- No auth required. Returns `{ status: "ok", version: "1.0.0" }`.

   - `POST /chat` -- Auth required. Reads JSON body. Forwards to `https://api.anthropic.com/v1/messages` with headers:
     - `Content-Type: application/json`
     - `x-api-key: c.env.ANTHROPIC_API_KEY`
     - `anthropic-version: 2023-06-01`
     Sets `stream: true` in the body. Returns the Anthropic response as SSE passthrough with headers `Content-Type: text/event-stream` and `Cache-Control: no-cache`.

   - `POST /stt` -- Auth required. Returns `{ message: "STT endpoint placeholder", status: "not_implemented" }` with status 501.

   - `POST /tts` -- Auth required. Returns `{ message: "TTS endpoint placeholder", status: "not_implemented" }` with status 501.

   **CORS middleware**: Use `hono/cors` with `origin: '*'` for development. Apply to all routes.

   **Input validation on /chat**: Before proxying, validate that the request body contains a `messages` array. If missing, return 400 `{ error: "messages array is required" }`.

   Export the Hono app as `export default app`.

5. Run `cd worker && npm install` to install dependencies.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> Worker | Desktop app sends requests to CF Worker over HTTPS. Token in header. |
| Worker -> Anthropic | Worker proxies to Anthropic API using server-side secret key. |
| Worker -> KV | Worker reads/writes rate limit counters to Cloudflare KV. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | worker/src/index.ts | mitigate | API keys stored as Wrangler secrets (D-12), never in source. Bindings type enforces runtime access only. |
| T-01-02 | Denial of Service | worker/src/index.ts | mitigate | Auth middleware validates x-app-token on all non-health routes. Token must be >= 32 chars. KV-based rate limiting at 60 req/min per token (D-10). |
| T-01-03 | Spoofing | worker/src/index.ts | accept | V1 token is UUID without cryptographic signing. Acceptable for private beta -- upgrade to HMAC-signed tokens in V2 if abuse detected. |
| T-01-04 | Tampering | Worker -> Anthropic | mitigate | HTTPS enforced by Cloudflare. Request body validated for required `messages` array before proxying. |
| T-01-05 | Information Disclosure | worker/src/index.ts | mitigate | CORS set to `origin: '*'` for dev. Must restrict to app origin before production release. |
</threat_model>
