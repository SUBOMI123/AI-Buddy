# Phase 1 — UI Design Contract

> Visual and interaction contract for the app shell overlay, system tray, and permission flows. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (SolidJS — shadcn is React-only) |
| Preset | not applicable |
| Component library | none — hand-rolled SolidJS components with CSS custom properties |
| Icon library | Lucide (tree-shakeable, available as `lucide-solid` for SolidJS) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` |

**Rationale:** SolidJS has no shadcn equivalent with the same registry model. The overlay UI surface is small (sidebar + input + list). Hand-rolled components with CSS custom properties that read OS theme via `prefers-color-scheme` is the correct weight for this phase. Lucide provides consistent 24px icons with SolidJS bindings.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding |
| sm | 8px | Input internal padding, list item gaps |
| md | 16px | Sidebar content padding (horizontal), section gaps |
| lg | 24px | Sidebar content padding (vertical top/bottom) |
| xl | 32px | Between major sections (input area vs guidance area) |
| 2xl | 48px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

Exceptions: Touch/click targets for tray menu items and the text input submit area must be minimum 44px tall for accessibility.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 16px | 600 (semibold) | 1.3 |

**3 sizes, 2 effective weights (400 and 600; 500 used only for labels).** 14px body is correct for a 300px-wide sidebar — 16px body would crowd the narrow panel. System font stack renders at native weight on each OS.

---

## Color

This phase uses OS-adaptive theming via `prefers-color-scheme` and Tauri's transparent window with platform vibrancy (macOS) / Mica (Windows). Colors are defined as CSS custom properties with light and dark variants.

| Role | Light Mode | Dark Mode | Usage |
|------|-----------|----------|-------|
| Dominant (60%) | transparent (OS vibrancy) | transparent (OS vibrancy) | Sidebar background — uses native blur material, not a solid color |
| Surface fallback | `rgba(255, 255, 255, 0.72)` | `rgba(28, 28, 30, 0.72)` | Fallback when vibrancy unavailable |
| Secondary (30%) | `rgba(0, 0, 0, 0.05)` | `rgba(255, 255, 255, 0.08)` | Input field background, dividers, list item hover |
| Text primary | `rgba(0, 0, 0, 0.88)` | `rgba(255, 255, 255, 0.92)` | Body text, guidance steps |
| Text secondary | `rgba(0, 0, 0, 0.50)` | `rgba(255, 255, 255, 0.50)` | Placeholder text, labels, step numbers |
| Accent (10%) | `#007AFF` | `#0A84FF` | Submit button icon, active shortcut indicator, permission grant button |
| Destructive | `#FF3B30` | `#FF453A` | Permission denied state only |

**Accent reserved for:** submit/send action in the input field, the "Grant Permission" button in the privacy disclosure, and the active state indicator on the system tray icon. Nothing else.

**Why rgba and not hex for surfaces:** The overlay is a transparent window with OS blur material. Solid hex backgrounds would defeat the vibrancy effect. All surface colors use alpha transparency so the OS material shows through.

## Component Inventory

Phase 1 renders exactly these components. No others.

| Component | Location | Interactive | Notes |
|-----------|----------|-------------|-------|
| SidebarShell | Overlay window | Yes (drag handle at top) | 300px fixed width, docked to screen edge, CSS `resize: none` |
| DragHandle | Top of SidebarShell | Yes (mousedown + drag) | 8px tall bar, centered grip indicator (3 horizontal lines), drag to reposition left/right edge |
| TextInput | Bottom of SidebarShell | Yes (type + submit) | Single-line input with placeholder, auto-focuses on panel open, submit via Enter or icon button |
| EmptyState | Center of SidebarShell | No | Shown when no guidance is active. Icon + heading + body text |
| GuidanceList | Center of SidebarShell | No (read-only) | Numbered steps rendered as ordered list. Placeholder for Phase 2 streaming content |
| PermissionDialog | Overlay window (modal) | Yes (grant/dismiss) | One-time screen capture permission flow with privacy disclosure |
| TrayMenu | OS-native tray | Yes (click items) | Native menu via Tauri TAO: Show/Hide, Preferences, Quit |

## Interaction Contracts

### Overlay Toggle (D-06, D-07, D-08)

| Trigger | Behavior |
|---------|----------|
| Global shortcut (default: Ctrl+Shift+Space on Windows, Cmd+Shift+Space on macOS) | Toggle sidebar visibility |
| Panel opens | Slide-in from docked edge, 200ms ease-out. Text input receives focus immediately. |
| Panel closes | Slide-out to docked edge, 150ms ease-in. Focus returns to previously active app window. |
| Click outside panel | Does NOT dismiss (user may click their work app). Dismiss only via shortcut or tray menu. |

### Sidebar Positioning (D-01)

| Behavior | Detail |
|----------|--------|
| Default dock | Right edge of primary display |
| Reposition | Drag handle allows moving to left or right edge only (vertical position: full height minus OS chrome) |
| Persistence | Docked edge (left/right) saved to local preferences, restored on next launch |
| Multi-monitor | Docks to the edge of the monitor where it was last positioned |

### Text Input (D-03, D-08)

| Behavior | Detail |
|----------|--------|
| Placeholder | "Ask me anything about what's on your screen..." (in text-secondary color) |
| Submit | Enter key or click send icon. In Phase 1, submission shows a "Not connected yet" inline message (no AI backend). |
| Disabled state | While permission dialog is showing, input is disabled with 50% opacity |

### Permission Flow (FOUND-04)

| Step | UI |
|------|-----|
| First launch | PermissionDialog appears inside the sidebar overlay (not a separate OS window) |
| Disclosure text | "AI Buddy captures a screenshot of your screen when you ask for help. Screenshots are sent to the AI for analysis and are never stored. No continuous recording." |
| Actions | "Grant Permission" (accent button) and "Not Now" (text link) |
| Grant → OS prompt | Triggers OS-level screen capture permission. On success, dialog dismissed, empty state shown. |
| Deny / Not Now | Dialog dismissed. Sidebar shows degraded empty state: "Screen capture permission needed to help you. Grant it in System Preferences > Privacy." |
| Permission already granted | Skip dialog entirely on subsequent launches |

### System Tray (FOUND-01)

| Item | Action |
|------|--------|
| Tray icon | Single-click toggles sidebar (same as global shortcut) |
| "Show AI Buddy" / "Hide AI Buddy" | Toggles sidebar visibility. Label reflects current state. |
| "Preferences..." | Reserved — opens preferences window (placeholder, not built in Phase 1) |
| Separator | Visual divider |
| "Quit AI Buddy" | Exits the process |

## Layout Specification

```
+--[ Primary Display ]------------------------------------------+
|                                                                |
|                                          +--[SidebarShell]--+ |
|                                          | [DragHandle]      | |
|   User's active application              |                   | |
|   (not obscured — sidebar docks          | [EmptyState]      | |
|    to the edge, not over content)        |  or               | |
|                                          | [GuidanceList]    | |
|                                          |  or               | |
|                                          | [PermissionDialog]| |
|                                          |                   | |
|                                          | [TextInput]       | |
|                                          +-------------------+ |
|                                          |<--- 300px ------->| |
+----------------------------------------------------------------+
```

- Sidebar height: 100% of screen height minus OS menu bar (macOS) or taskbar (Windows)
- Sidebar width: 300px fixed, not resizable
- Content area: fills between DragHandle (top) and TextInput (bottom), scrollable if guidance overflows
- Border: 1px `rgba(0,0,0,0.12)` (light) / `rgba(255,255,255,0.12)` (dark) on the inner edge only (the edge facing user's workspace)

## Animation

| Animation | Duration | Easing | Property |
|-----------|----------|--------|----------|
| Sidebar open | 200ms | `cubic-bezier(0.16, 1, 0.3, 1)` | `translateX` from off-screen to 0 |
| Sidebar close | 150ms | `cubic-bezier(0.4, 0, 1, 1)` | `translateX` from 0 to off-screen |
| Input focus ring | 150ms | ease | `box-shadow` (accent color at 30% opacity, 0 0 0 3px) |

No other animations in Phase 1. No loading spinners (no async operations in this phase beyond permission check).

## CSS Custom Properties Contract

All visual tokens defined as CSS custom properties on `:root`. Theme switching handled by `prefers-color-scheme` media query.

```css
:root {
  --sidebar-width: 300px;
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
  --font-size-label: 12px;
  --font-size-body: 14px;
  --font-size-heading: 16px;
  --font-weight-regular: 400;
  --font-weight-medium: 500;
  --font-weight-semibold: 600;
  --line-height-body: 1.5;
  --line-height-label: 1.4;
  --line-height-heading: 1.3;
  --radius-sm: 6px;
  --radius-md: 10px;
  --radius-lg: 14px;
  --transition-fast: 150ms ease;
  --transition-normal: 200ms ease;
}

@media (prefers-color-scheme: light) {
  :root {
    --color-surface: rgba(255, 255, 255, 0.72);
    --color-surface-secondary: rgba(0, 0, 0, 0.05);
    --color-text-primary: rgba(0, 0, 0, 0.88);
    --color-text-secondary: rgba(0, 0, 0, 0.50);
    --color-accent: #007AFF;
    --color-destructive: #FF3B30;
    --color-border: rgba(0, 0, 0, 0.12);
  }
}

@media (prefers-color-scheme: dark) {
  :root {
    --color-surface: rgba(28, 28, 30, 0.72);
    --color-surface-secondary: rgba(255, 255, 255, 0.08);
    --color-text-primary: rgba(255, 255, 255, 0.92);
    --color-text-secondary: rgba(255, 255, 255, 0.50);
    --color-accent: #0A84FF;
    --color-destructive: #FF453A;
    --color-border: rgba(255, 255, 255, 0.12);
  }
}
```
