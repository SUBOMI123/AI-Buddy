    ```typescript
    app.post('/stripe-webhook', async (c) => {
      const signature = c.req.header('stripe-signature');
      if (!signature) return c.text('', 400);

      const body = await c.req.text();

      const stripe = new Stripe(c.env.STRIPE_SECRET_KEY, {
        httpClient: Stripe.createFetchHttpClient(),
      });

      let event: Stripe.Event;
      try {
        event = await stripe.webhooks.constructEventAsync(
          body,
          signature,
          c.env.STRIPE_WEBHOOK_SECRET,
          undefined,
          Stripe.createSubtleCryptoProvider(),
        );
      } catch {
        return c.text('Webhook signature verification failed', 400);
      }

      if (event.type === 'checkout.session.completed') {
        const session = event.data.object as Stripe.Checkout.Session;
        const uuid = session.metadata?.installation_uuid;
        if (uuid && c.env.RATE_LIMIT) {
          await c.env.RATE_LIMIT.put(`subscription:${uuid}`, 'active');
        }
      }

      if (event.type === 'customer.subscription.deleted') {
        const sub = event.data.object as Stripe.Subscription;
        const uuid = sub.metadata?.installation_uuid;
        if (uuid && c.env.RATE_LIMIT) {
          await c.env.RATE_LIMIT.put(`subscription:${uuid}`, 'cancelled');
        }
      }

      return c.text('', 200);
    });
    ```

      if (c.env.RATE_LIMIT) {

      }

      return c.json({ status: isActive ? 'active' : 'free' });

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stripe → Worker (/stripe-webhook) | Stripe signs webhook payloads; Worker must verify before trusting event data |
| App → Worker (/create-checkout, /refresh-subscription) | HMAC-signed x-app-token proves app identity; tokenValue (UUID) is the user discriminator |
| KV read on subscription:uuid | Worker trusts KV value — if KV is compromised, quota bypass is possible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | /stripe-webhook | mitigate | Validate Stripe-Signature with `stripe.webhooks.constructEventAsync()` + `Stripe.createSubtleCryptoProvider()`; return 400 immediately on verification failure — never process unverified events |
| T-13-02 | Elevation of Privilege | subscription:uuid KV key | mitigate | Key uses installation UUID (not user-controlled input in the header — extracted from HMAC-verified token, not from request body); quota bypass can only be granted by a verified Stripe webhook or /refresh-subscription calling real Stripe API |
| T-13-03 | Spoofing | /create-checkout uuid body param | mitigate | UUID in POST body is validated to be non-empty string; critical: the subscription is written keyed on the UUID from the *verified token* (c.get('tokenValue')), not the body uuid — body uuid is only passed to Stripe metadata. Webhook writes `subscription:${session.metadata.installation_uuid}` which the attacker controls only if they can forge a Stripe webhook (mitigated by T-13-01) |
| T-13-04 | Information Disclosure | /quota endpoint | accept | Returns only count data for the authenticated token's own quotas; no cross-user leakage — key is `quota:*:${tokenValue}` where tokenValue comes from verified token |
| T-13-05 | Denial of Service | checkQuota() TOCTOU | accept | Same race condition accepted in Phase 12 for rate limiting; KV eventual consistency allows minor bursts; Durable Objects would fix this but are out of scope for beta traffic |
| T-13-06 | Repudiation | /stripe-webhook KV write | accept | Webhook events are logged by Stripe Dashboard; no additional audit trail required for beta |
| T-13-07 | Tampering | /refresh-subscription Stripe query | mitigate | Query uses `metadata['installation_uuid']:'${uuid}'` where uuid comes from HMAC-verified token — attacker cannot elevate by passing a different UUID; Stripe metadata is set by Worker at checkout creation |
</threat_model>
