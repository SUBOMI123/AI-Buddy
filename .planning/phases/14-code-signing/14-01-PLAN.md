<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Config files → Build system | entitlements.plist and tauri.conf.json drive signing behavior; over-granting or wrong values affect what the signed app can do |
| Info.plist → macOS TCC | NSScreenCaptureUsageDescription is what macOS shows users in the permission dialog; inaccurate text is a trust violation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01-01 | Elevation of Privilege | entitlements.plist | mitigate | Use only the 5 documented minimum entitlements; task action explicitly forbids adding additional keys beyond these 5 |
| T-14-01-02 | Information Disclosure | tauri.conf.json | accept | signingIdentity value is the certificate name (not a secret); passwords never appear in config files per D-03 |
| T-14-01-03 | Spoofing | Info.plist NSScreenCaptureUsageDescription | mitigate | Description text must accurately describe what captures are used for; approved text from CONTEXT.md D-05 is used verbatim |
</threat_model>
