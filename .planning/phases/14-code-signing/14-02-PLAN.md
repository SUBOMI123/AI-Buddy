  <what-to-do>
Complete these steps in order. Every step is required before the build can run.

**Step 1 — Confirm Apple Developer Program membership**
Visit https://developer.apple.com/account — if you see "You are not enrolled", enroll now ($99/yr). You cannot create a Developer ID certificate without an active membership.

**Step 2 — Create Developer ID Application certificate**
1. Go to https://developer.apple.com/account/resources/certificates/add
2. Select "Developer ID Application" (under the Software section)
3. Follow the Certificate Signing Request (CSR) flow:
   - Open Keychain Access.app → Certificate Assistant → Request a Certificate from a Certificate Authority
   - Enter your Apple ID email, leave CA Email blank, select "Saved to disk"
   - Save the `.certSigningRequest` file to Desktop
4. Upload the `.certSigningRequest` file on the Apple Developer portal
5. Download the resulting `.cer` file
6. Double-click the `.cer` file — it installs into Keychain automatically

**Step 3 — Verify certificate is installed**
Run this command and confirm it prints a line containing "Developer ID Application":
```bash
security find-identity -v -p codesigning
```
Expected output line format: `"Developer ID Application: Your Name (XXXXXXXXXX)"`
Copy the entire string in double-quotes — you will need it in the next step.

**Step 4 — Generate app-specific password**
1. Go to https://appleid.apple.com → Sign In → Security section
2. Click "Generate Password" under "App-Specific Passwords"
3. Label it "ai-buddy-notarize"
4. Copy the generated password (format: `xxxx-xxxx-xxxx-xxxx`)

**Step 5 — Store notarytool Keychain profile**
Replace placeholders with your actual values:
```bash
xcrun notarytool store-credentials "ai-buddy-notarize" \
  --apple-id "you@youremail.com" \
  --team-id "XXXXXXXXXX" \
  --password "xxxx-xxxx-xxxx-xxxx"
```
The Team ID is the 10-character string in parentheses from Step 3.
The password is the app-specific password from Step 4.

**Verification before resuming:**
- `security find-identity -v -p codesigning` shows at least 1 valid identity with "Developer ID Application"
- No error when running: `xcrun notarytool history --keychain-profile "ai-buddy-notarize"` (may return empty list — that's fine)

**Provide Claude with:**
The full signing identity string (e.g., `Developer ID Application: Jane Smith (AB12CD34EF)`)

**Part B — Create scripts/sign-and-notarize.sh**

Create the `scripts/` directory and write `scripts/sign-and-notarize.sh` with this content:

```bash
#!/usr/bin/env bash
# scripts/sign-and-notarize.sh
# Phase 14: Manual sign + notarize + staple pipeline for AI Buddy macOS builds.
#
# Usage (Phase 14 — local, uses Keychain profile):
#   export APPLE_SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID)"
#   export APPLE_ID="you@email.com"
#   export APPLE_PASSWORD="xxxx-xxxx-xxxx-xxxx"   # app-specific password
#   export APPLE_TEAM_ID="XXXXXXXXXX"
#   bash scripts/sign-and-notarize.sh
#
# Phase 15 CI note:
#   Replace APPLE_PASSWORD with $APPLE_APP_SPECIFIC_PASSWORD from GitHub Secrets.
#   KEYCHAIN_PROFILE env var is not used by cargo tauri build — env vars above are sufficient.
#   The Keychain profile (ai-buddy-notarize) is a local-only convenience; CI uses env vars directly.
#
# Requires: Xcode CLI tools, cargo tauri 2.10+, Developer ID Application cert in Keychain.

set -euo pipefail

# Validate required env vars
: "${APPLE_SIGNING_IDENTITY:?Set APPLE_SIGNING_IDENTITY before running}"
: "${APPLE_ID:?Set APPLE_ID before running}"
: "${APPLE_PASSWORD:?Set APPLE_PASSWORD before running}"
: "${APPLE_TEAM_ID:?Set APPLE_TEAM_ID before running}"

echo "==> Building, signing, and notarizing AI Buddy..."
echo "    Identity: $APPLE_SIGNING_IDENTITY"
echo "    Team ID:  $APPLE_TEAM_ID"
echo "    Apple ID: $APPLE_ID"
echo ""

# Step 1: Build (Tauri auto-signs and auto-notarizes when APPLE_* vars are set)
echo "==> Running cargo tauri build..."
cargo tauri build

# Step 2: Find the DMG (architecture-specific filename)
DMG_PATH=$(ls target/release/bundle/macos/*.dmg 2>/dev/null | head -1)
if [ -z "$DMG_PATH" ]; then
  echo "ERROR: No DMG found in target/release/bundle/macos/. Build may have failed."
  exit 1
fi
echo "==> Found DMG: $DMG_PATH"

# Step 3: Staple the notarization ticket to the DMG for offline Gatekeeper (SIGN-02)
# Note: cargo tauri build auto-notarizes but does NOT staple. Staple is always a manual step.
echo "==> Stapling notarization ticket..."
xcrun stapler staple "$DMG_PATH"

# Step 4: Validate staple succeeded
echo "==> Validating staple..."
xcrun stapler validate "$DMG_PATH"

# Step 5: Verify Gatekeeper accepts the .app inside the DMG
APP_PATH="target/release/bundle/macos/AI Buddy.app"
if [ -d "$APP_PATH" ]; then
  echo "==> Verifying codesign integrity..."
  codesign --verify --deep --strict --verbose=2 "$APP_PATH"

  echo "==> Verifying Gatekeeper acceptance (spctl)..."
  spctl --assess --type execute --verbose=4 "$APP_PATH"

  echo "==> Verifying entitlements in signed bundle..."
  codesign --display --entitlements - "$APP_PATH"
else
  echo "WARNING: .app not found at expected path for verification. Check DMG manually."
fi

echo ""
echo "==> Done. Artifacts:"
echo "    DMG (stapled): $DMG_PATH"
echo ""
echo "==> Phase 14 offline test (SIGN-02):"
echo "    1. Mount the DMG: open \"$DMG_PATH\""
echo "    2. Drag AI Buddy.app to /Applications"
echo "    3. Disable WiFi"
echo "    4. Double-click AI Buddy in /Applications — must launch with no Gatekeeper dialog"
echo ""
echo "==> Phase 15 handoff note:"
echo "    CI replaces Keychain profile with env vars: APPLE_APP_SPECIFIC_PASSWORD"
echo "    Same APPLE_* env vars work in GitHub Actions secrets."
```

Make the script executable:
```bash
chmod +x scripts/sign-and-notarize.sh
```

CRITICAL: Do NOT write any credential values (passwords, Apple IDs, Team IDs) into the script. The script uses env vars only. The Keychain profile (ai-buddy-notarize) is referenced in comments only — the actual build uses the APPLE_PASSWORD env var.

**Credential model (per D-03):** The Keychain profile `ai-buddy-notarize` (created in Task 1) is the durable credential store — it holds your Apple ID, Team ID, and app-specific password encrypted in macOS Keychain. For the build, credentials are injected as session-scoped shell env vars that evaporate when the terminal closes. They are never written to disk or committed.

**Step 1 — Export credentials as session-scoped env vars in your terminal**

Look up your values from the Keychain profile you created in Task 1. The signing identity came from `security find-identity`; the Apple ID, Team ID, and password are what you entered into `notarytool store-credentials`.

```bash
export APPLE_SIGNING_IDENTITY="Developer ID Application: Your Name (XXXXXXXXXX)"
export APPLE_ID="you@youremail.com"
export APPLE_PASSWORD="xxxx-xxxx-xxxx-xxxx"   # app-specific password (NOT your Apple ID password)
export APPLE_TEAM_ID="XXXXXXXXXX"              # 10-char string from Keychain identity
```

These env vars exist only for the duration of this terminal session. When you close the terminal, they are gone.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terminal env vars → cargo tauri build | APPLE_PASSWORD is in shell environment; must not be echoed to logs or written to files |
| Keychain profile → notarytool | ai-buddy-notarize profile stores Apple ID credentials encrypted in macOS Keychain; never exported |
| Apple notary service → DMG artifact | Apple cryptographically signs the notarization ticket; staple embeds it in the DMG binary |
| GitHub Release → beta user download | Signed DMG must come from official GitHub Release URL; SmartScreen docs must instruct this |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-02-01 | Information Disclosure | Shell env vars (APPLE_PASSWORD) | mitigate | Script uses `set -euo pipefail`; env vars are export-only, never written to disk or echoed in plaintext; app-specific password can be revoked independently if exposed |
| T-14-02-02 | Information Disclosure | Keychain profile "ai-buddy-notarize" | mitigate | Profile stored in macOS Keychain (encrypted at rest); not exported to any file; CI (Phase 15) will use GitHub Secrets env vars instead |
| T-14-02-03 | Spoofing | sign-and-notarize.sh credentials | mitigate | Script accepts only env vars for credentials; acceptance criteria verify no literal credential strings are written into the script file |
| T-14-02-04 | Tampering | DMG artifact post-notarization | accept | Apple's notarization ticket is cryptographically bound to the DMG hash; any modification after staple breaks the ticket and Gatekeeper rejects the app |
| T-14-02-05 | Elevation of Privilege | Developer ID Application certificate | mitigate | Certificate provisioned via Apple's CSR process; .p12 not exported to disk in Phase 14; private key stays in Keychain only |
</threat_model>
