```rust
pub fn open_db(app: &AppHandle) -> SqlResult<Connection> {
    let dir = app
        .path()
        .app_data_dir()
        .expect("Failed to get app data dir");
    std::fs::create_dir_all(&dir).ok();
    let db_path = dir.join("memory.db");
    let conn = Connection::open(&db_path)?;
    // Set file permissions to 0600 (owner-only) on Unix — matches preferences.rs pattern
    #[cfg(unix)]
    {
        use std::os::unix::fs::PermissionsExt;
        let _ = std::fs::set_permissions(
            &db_path,
            std::fs::Permissions::from_mode(0o600),
        );
    }
    init_schema(&conn)?;
    Ok(conn)
}
```

**Schema** (two tables — D-09: no vector columns needed for v1):
```rust
pub fn init_schema(conn: &Connection) -> SqlResult<()> {
    conn.execute_batch("
        CREATE TABLE IF NOT EXISTS interactions (
            id          INTEGER PRIMARY KEY AUTOINCREMENT,
            task_label  TEXT    NOT NULL,
            raw_intent  TEXT    NOT NULL,
            app_context TEXT,
            guidance    TEXT    NOT NULL,
            tier        INTEGER NOT NULL DEFAULT 1,
            timestamp   INTEGER NOT NULL
        );
        CREATE INDEX IF NOT EXISTS idx_interactions_task_label
            ON interactions(task_label);

        CREATE TABLE IF NOT EXISTS task_encounters (
            task_label      TEXT    PRIMARY KEY,
            encounter_count INTEGER NOT NULL DEFAULT 1,
            first_seen      INTEGER NOT NULL,
            last_seen       INTEGER NOT NULL
        );
    ")?;
    Ok(())
}
```

**Private helper — epoch timestamp:**
```rust
fn now_unix() -> i64 {
    SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap_or_default()
        .as_secs() as i64
}
```

**Private helper — classify_intent** (async reqwest call to Worker /classify, D-02):
- Takes `&AppHandle` and `raw_intent: &str`
- Posts `{ "intent": raw_intent }` to `{WORKER_URL}/classify` with `x-app-token` header (use `preferences::cmd_get_token` logic — call `preferences::get_installation_token` then sign)
- On success: returns the `label` field from JSON response `{ "label": "..." }`
- On any error (network, parse, non-200): return deterministic fallback = first 3 words of raw_intent lowercased, whitespace replaced with `_`, truncated to 50 chars (e.g. `"how do I export this"` → `"how_do_i"`)
- Use `reqwest::Client::new()` — reqwest is already in Cargo.toml
- The WORKER_URL must be read from an env var `WORKER_URL` at runtime (use `std::env::var("WORKER_URL").unwrap_or_else(|_| "http://localhost:8787".to_string())`)
- Note: DO NOT hold any Mutex lock when calling this function — it must be called before acquiring the MemoryDb lock (per Pitfall 1 in RESEARCH.md)

**cmd_prepare_guidance_context** (called BEFORE streamGuidance to get tier — D-03):
```rust
#[derive(Serialize)]
pub struct GuidanceContext {
    pub tier: u8,
    pub task_label: String,
    pub encounter_count: u32,
}

#[tauri::command]
pub async fn cmd_prepare_guidance_context(
    app: AppHandle,
    db: State<'_, MemoryDb>,
    raw_intent: String,
) -> Result<GuidanceContext, String> {
    // 1. Classify intent BEFORE acquiring lock (critical — avoid Mutex held during HTTP)
    let task_label = classify_intent(&app, &raw_intent).await;
    // 2. Now lock briefly to read encounter count
    let count = {
        let conn = db.0.lock().map_err(|e| e.to_string())?;
        conn.query_row(
            "SELECT encounter_count FROM task_encounters WHERE task_label = ?1",
            params![task_label],
            |row| row.get::<_, u32>(0),
        ).unwrap_or(0)
    };
    let tier = match count {
        0 => 1,
        1 => 2,
        _ => 3,
    };
    Ok(GuidanceContext { tier, task_label, encounter_count: count })
}
```

**cmd_record_interaction** (called AFTER onDone, D-01):
- Async command — annotated `async fn`
- Accepts: `app: AppHandle`, `db: State<'_, MemoryDb>`, `task_label: String`, `raw_intent: String`, `app_context: Option<String>`, `guidance: String`, `tier: u8`
- All string fields bound as SQLite params — never string-interpolated (T-05-01 STRIDE mitigation)
- INSERT into `interactions` with `params![task_label, raw_intent, app_context, guidance, tier, now_unix()]`
- UPSERT into `task_encounters` using ON CONFLICT syntax (NOT INSERT OR REPLACE — that resets counter):
```sql
INSERT INTO task_encounters (task_label, encounter_count, first_seen, last_seen)
VALUES (?1, 1, ?2, ?2)
ON CONFLICT(task_label) DO UPDATE SET
  encounter_count = encounter_count + 1,
  last_seen = excluded.last_seen
```
- Lock must be acquired ONCE after the INSERT-then-UPSERT in a single lock scope (both ops, then drop)
- Returns `Result<(), String>`

**cmd_get_memory_context** (returns summary string for system prompt injection, D-08):
- Sync command (no HTTP call needed)
- Derives a summary string from task_encounters — cap to 5 tasks per category, total string must stay under 200 tokens (~800 chars)
- Format: `"User skill context: Has completed [export_pdf] 4x, [insert_table] 2x. Recurring struggles: [create_pivot_table] (3 attempts)."`
- Use LIMIT 5 on queries to prevent unbounded growth
- Returns `Result<String, String>`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw_intent → SQLite | User-controlled string flows into DB INSERT — must be parameterised |
| task_label → SQLite | Claude-classified label flows into upsert — still parameterised (label sanitised but not trusted) |
| /classify intent → Anthropic | raw_intent forwarded to Anthropic API via Worker — length-capped at 500 chars |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | memory.rs INSERT/UPSERT | mitigate | All string values bound as rusqlite `params![]` — never string-interpolated. Covers SQL injection via raw_intent and task_label. |
| T-05-02 | Tampering | worker /classify input | mitigate | Validate `intent` is string, length <= 500 chars, return 400 on violation before Anthropic call. |
| T-05-03 | Information Disclosure | cmd_get_memory_context | mitigate | D-08 enforced: only aggregate counts and labels in summary string. Raw guidance text and raw_intent never included. LIMIT 5 per category caps exposure. |
| T-05-04 | Information Disclosure | memory.db file | mitigate | File permissions set to 0600 (owner-only) on creation, matching preferences.rs pattern. Windows relies on AppData directory ACLs. |
| T-05-05 | Denial of Service | memory context string growth | accept | Mitigated by LIMIT 5 in queries and ~800 char cap on output string. No unbounded growth path remains. |
</threat_model>
