<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local machine → Cloudflare API | Wrangler CLI OAuth — credentials stored in ~/.wrangler/, not in repo |
| wrangler.toml → git | KV namespace ID committed to git — this is safe (not a secret) |
| Wrangler secrets → Cloudflare | API keys transmitted over HTTPS to Cloudflare secret storage — never stored locally or in git |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-06 | Information Disclosure | APP_HMAC_SECRET in wrangler secret put | mitigate | Secret set only via CLI prompt — never written to a file or terminal history; use `wrangler secret put` (interactive) not `echo value | wrangler secret put` |
| T-12-07 | Information Disclosure | API keys (ANTHROPIC, ASSEMBLYAI, ELEVENLABS) | mitigate | Set via `wrangler secret put` — encrypted at rest in Cloudflare, never in wrangler.toml or source |
| T-12-08 | Spoofing | APP_HMAC_SECRET mismatch between Worker and app | mitigate | Explicit instructions to use same value in both wrangler secret and .cargo/config.toml; mismatch causes 401 on every request (immediately detectable) |
| T-12-09 | Information Disclosure | KV namespace ID in wrangler.toml committed to git | accept | Namespace ID is not a secret — it's a resource identifier equivalent to a database name; the KV data itself is protected by auth middleware |
| T-12-10 | Denial of Service | Unauthenticated /health endpoint | accept | /health returns only static {"status":"ok"} — no data disclosure, no compute cost; standard health check pattern |
</threat_model>
