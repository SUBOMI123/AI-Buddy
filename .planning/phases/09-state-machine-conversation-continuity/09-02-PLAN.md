  ```typescript
  export interface SessionExchange {
    intent: string;   // user's raw text for this turn
    guidance: string; // complete guidance text (set after onDone fires)
  }
  ```

  **Props interface:**
  ```typescript
  interface SessionFeedProps {
    sessionHistory: SessionExchange[];      // prior completed exchanges
    streamingText: string;                  // active (current) exchange in progress
    ttsEnabled?: boolean;                   // show TTS play buttons
    ref?: (el: HTMLDivElement) => void;     // parent needs ref for auto-scroll
  }
  ```

  **Component structure:**

  The component is a scrollable container `div` with:
  - `ref` callback: assign to the `ref` prop so the parent (SidebarShell) can call `el.scrollTop = el.scrollHeight`
  - `flex: 1; overflow-y: auto; display: flex; flex-direction: column; gap: var(--space-sm)`
  - `aria-label="Conversation history"`

  Inside: use `<For each={props.sessionHistory}>` to render each prior exchange as a "prior exchange item", then render the active exchange below all prior ones.

  **Prior exchange item layout** (per UI-SPEC.md — D-03, D-04):
  Each item is a `div` with `padding: var(--space-xs) 0`:
  1. User intent label: `<p>` with `font-size: var(--font-size-label)`, `line-height: var(--line-height-label)`, `color: var(--color-text-secondary)`, `padding-bottom: var(--space-xs)`, `margin: 0`. Content: `exchange.intent` (verbatim, no truncation for history labels — truncation is only for the task header).
  2. Guidance lines: split `exchange.guidance` by `\n`, filter empty, render each line as a `<p>` with `font-size: var(--font-size-body)`, `line-height: var(--line-height-body)`, `color: var(--color-text-secondary)`, `margin: 0`, `white-space: pre-wrap`, `word-wrap: break-word`.

  No dividers between items (D-04). No TTS buttons on prior exchanges (they are historical, playback on historical items is Phase 10 concern).

  **Active exchange item** (only shown when `props.streamingText` is non-empty):
  Use `<Show when={props.streamingText.length > 0}>`. Split `props.streamingText` by `\n`, filter empty, render each line as a `<p>` with `font-size: var(--font-size-body)`, `line-height: var(--line-height-body)`, `color: var(--color-text-primary)`, `margin: 0`, `white-space: pre-wrap`, `word-wrap: break-word`.

  Per GuidanceList pattern, each active line also gets the optional TTS Volume2 button (same style as GuidanceList — visible only when `props.ttsEnabled`). Import `Volume2` from `lucide-solid` and `playTts` from `../lib/tauri`.

  **No intent label for the active exchange** — the TaskHeaderStrip in SidebarShell already displays it (per UI-SPEC.md Active exchange item spec).

  ```typescript
  import { For, Show } from "solid-js";

  ```

  The component does NOT import `createSignal` — it is a pure props-driven renderer. All reactive state lives in SidebarShell.
