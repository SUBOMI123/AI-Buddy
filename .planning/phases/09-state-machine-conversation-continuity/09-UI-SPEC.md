## Component Inventory

### TaskHeaderStrip

A new atomic UI element that appears/disappears as a unit (CONTEXT.md "Specific Ideas").

| Property | Value |
|----------|-------|
| Visibility | Shown when `contentState !== "empty"` AND session has a `lastIntent`. Hidden in idle/empty state (D-07). |
| Position | Below the settings icon row (`border-bottom: 1px solid var(--color-border)`), above the scrollable feed. `flex-shrink: 0`. |
| Background | `var(--color-surface-secondary)` |
| Border | `border-bottom: 1px solid var(--color-border)` |
| Padding | `var(--space-sm) var(--space-md)` (8px 16px) |

**Task header text:**
- Content: raw first intent of session, truncated at 50 characters with trailing `…` (D-06)
- Font: 12px / weight 400 / line-height 1.4 (`--font-size-label`)
- Color: `var(--color-text-primary)`
- Display: single line, `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`

**"New task" link:**
- Content: `New task`
- Element: `<button>` styled as inline text link (no border, no background)
- Font: 12px / weight 400 (`--font-size-label`)
- Color: `var(--color-accent)`
- Text decoration: `underline`
- Touch target: `min-height: 44px; display: inline-flex; align-items: center`
- Padding: `0`
- Placement: directly below the task header text, same column (not inline — separate line)

**Layout:**
```
[TaskHeaderStrip]
  ├── task header text  (color-text-primary, 12px, truncated)
  └── "New task" link   (color-accent, 12px, underlined, below header text)
```

### SessionFeed

Replaces/wraps the current `GuidanceList` to render an array of `{intent, guidance}` exchange objects.

| Property | Value |
|----------|-------|
| Container | `flex: 1; overflow-y: auto; display: flex; flex-direction: column; gap: var(--space-sm)` |
| Scroll behavior | Auto-scroll to bottom when new response starts streaming (D-05). Use `scrollTop = scrollHeight` on the container ref. Trigger in `onToken` callback when `contentState` transitions from `"loading"` to `"streaming"`. |

**Prior exchange item (all exchanges except the active one):**

| Sub-element | Value |
|-------------|-------|
| User intent label | 12px / 400 / 1.4 / `var(--color-text-secondary)` / padding-bottom: `var(--space-xs)` |
| Guidance text | 14px / 400 / 1.5 / `var(--color-text-secondary)` |
| Item padding | `var(--space-xs) 0` top/bottom |
| No divider | No border between items (D-04) |

**Active exchange item (last item — currently streaming or completed):**

| Sub-element | Value |
|-------------|-------|
| User intent label | Not shown for the active exchange — task header strip already displays it |
| Guidance text | 14px / 400 / 1.5 / `var(--color-text-primary)` |
| Behavior | Streams in via existing `streamingText` signal; becomes a frozen prior exchange when `contentState` returns to `"empty"` or new intent submitted |

## Interaction Contracts

### State Machine (D-11)

`onOverlayShown` must only reset transient states:

| Signal | Resets on overlay show? |
|--------|------------------------|
| `contentState` | Only if currently `"loading"`, `"streaming"`, or `"error"` — set to `"empty"`. If already `"empty"`, no change. |
| `errorMessage` | Yes — always clear |
| `streamingText` | Only if contentState was transient — clear with it |
| `sttError` | Yes — always clear |
| `abortController` | Yes — abort and null |
| `sessionHistory` (new) | No — persists across hide/show |
| `lastIntent` (task header) | No — persists across hide/show |
| `detectedApp` | Yes — reset to null, fire new `getActiveApp()` (existing behavior) |

### "New task" Reset (D-01, D-02)

On click:
1. `sessionHistory` → `[]`
2. `lastIntent` → `""`
3. `streamingText` → `""`
4. `contentState` → `"empty"`
5. `errorMessage` → `""`
6. `abortController?.abort()` → null
7. Focus input field

No confirmation dialog. No destructive warning. The action is low-stakes and reversible by re-submitting.

### Session History Cap (D-09)

When adding a completed exchange to history:
- If `sessionHistory.length >= 3`, remove oldest entry before appending new one.
- Cap at 3 prior turns (6 messages: 3 user + 3 assistant) passed to Claude.

### Auto-scroll (D-05)

Trigger: when `contentState` transitions from `"loading"` → `"streaming"` (first token received).
Method: `containerRef.scrollTop = containerRef.scrollHeight`
Condition: unconditional — always scroll to bottom on new response start.

## Copywriting Contract

|---------|------|
| Primary CTA | "New task" (text link — resets session) |

| Destructive confirmation | Not applicable — "New task" requires no confirmation (low-stakes, in-memory only) |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| "New task" button | `aria-label="Start a new task"` |
| Task header | `aria-live="polite"` on the strip so screen readers announce when task context changes |
| Session feed | `aria-label="Conversation history"` on the scroll container |
| Min touch target | 44px min-height on all interactive elements (existing pattern, carried forward) |
| Color contrast | `var(--color-text-secondary)` at 0.50 opacity on white/dark backgrounds — marginal at WCAG AA. Acceptable for secondary metadata; not used for actionable elements. |
