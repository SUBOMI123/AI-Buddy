<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Enable GitHub Pages in repo Settings</name>
  <what-to-do>
Enable GitHub Pages for the AI-Buddy repo so that docs/index.html (already committed from plan 16-01) is served at https://subomi123.github.io/AI-Buddy/.

Exact steps:
1. Go to https://github.com/SUBOMI123/AI-Buddy/settings/pages
2. Under "Build and deployment" → "Source", select **"Deploy from a branch"**
3. Under "Branch", select **"main"** from the branch dropdown
4. Under the folder dropdown (next to the branch), select **"/docs"**
5. Click **"Save"**
6. GitHub will show a banner: "GitHub Pages source saved." and begin building
7. Wait 1–3 minutes for the first build to complete
8. Refresh the Settings → Pages screen — you should see a green banner with the URL: https://subomi123.github.io/AI-Buddy/

Note: If the repo is private, GitHub Pages requires a GitHub Pro, Team, or Enterprise account. If the step above fails with a plan/billing error, make the repo public first (Settings → General → Danger Zone → Change repository visibility → Public).
  </what-to-do>
  <resume-signal>Type "done" after clicking Save and seeing the green Pages URL banner, or describe what you see if the setting is not available.</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Verify GitHub Pages is live</name>
  <files></files>
  <action>
After the human enables GitHub Pages (Task 1), verify the site is actually serving by checking the URL returns HTTP 200 and contains expected content.

Run:
```bash
curl -s -o /dev/null -w "%{http_code}" https://subomi123.github.io/AI-Buddy/
```

Expected: `200`

If you get `404`: GitHub Pages build may still be in progress — wait 2 minutes and retry (up to 3 attempts with 60-second waits). If still 404 after 6 minutes, report the failure.

Also confirm page content contains expected text:
```bash
curl -s https://subomi123.github.io/AI-Buddy/ | grep -c "AI Buddy"
```

Expected: a number greater than 0 (the page contains "AI Buddy" in multiple places).

If the HTTP check passes but content check fails (returns 0), the docs/index.html file may not have been committed in plan 16-01. Check that docs/index.html is in the git history on main branch:
```bash
git -C /Users/subomi/Desktop/AI-Buddy log --oneline -- docs/index.html | head -3
```
