<task type="auto">
  <name>Task 2: Create .env.example documenting unified WORKER_URL config</name>
  <files>.env.example</files>
  <action>
Create `.env.example` at the repository root. This file is the canonical reference for all environment variables required to build and run AI Buddy.

The two WORKER_URL variables serve different consumers:
- `VITE_WORKER_URL` — read by Vite at build time via `import.meta.env.VITE_WORKER_URL` in `src/lib/ai.ts`. Required for the SolidJS frontend to know the Worker endpoint.
- `WORKER_URL` — read by the Rust build at compile time via `option_env!("WORKER_URL")` in `src-tauri/src/shortcut.rs` and `src-tauri/src/voice/tts.rs`, and at runtime via `std::env::var("WORKER_URL")` in `src-tauri/src/memory.rs`. Required for all Rust Worker calls.

Both must point to the same Worker URL in production. In development, both default to `http://localhost:8787` when absent — this default is baked into each consumer's fallback.

File content:

```
# AI Buddy — Environment Variables
# Copy this file to .env and fill in values before running.
# IMPORTANT: .env is gitignored. Never commit actual values.

# ============================================================
# Worker URL — MUST be set in both variables for a production
# build. Both consumers fall back to http://localhost:8787
# when absent (development default).
#
# Frontend consumer: src/lib/ai.ts (import.meta.env.VITE_WORKER_URL)
# Rust consumers:    src-tauri/src/shortcut.rs (option_env!)
#                    src-tauri/src/voice/tts.rs (option_env!)
#                    src-tauri/src/memory.rs    (std::env::var)
#
# For production builds, set BOTH to the deployed Worker URL:
#   https://ai-buddy-proxy.<your-account>.workers.dev
# ============================================================
VITE_WORKER_URL=http://localhost:8787
WORKER_URL=http://localhost:8787

# ============================================================
# These are set as Wrangler secrets, NOT in this file.
# Set them via:
#   npx wrangler secret put ANTHROPIC_API_KEY
#   npx wrangler secret put ASSEMBLYAI_API_KEY
#   npx wrangler secret put ELEVENLABS_API_KEY
#   npx wrangler secret put APP_HMAC_SECRET
# ============================================================
```

Also verify that `.env` and `.env.local` are already in `.gitignore`. If not, add them. Do not modify the actual `.env` file if one exists — only create `.env.example`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebView → Cloudflare Worker | SolidJS frontend fetch calls cross from Tauri WebView to the Worker; CORS controls which origins the browser allows |
| Rust backend → Cloudflare Worker | Rust reqwest calls are not subject to CORS (not a browser context) |
| Developer machine → Production deploy | wrangler.toml and tauri.conf.json placeholders are the only gate before secrets/endpoints go live |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Spoofing | CORS origin in worker/src/index.ts | mitigate | Restrict allowed origins to exactly `['http://localhost:1420', 'tauri://localhost']` — no wildcard. Task 1 implements this. |
| T-07-02 | Information Disclosure | .env.example committed to repo | accept | .env.example contains only placeholder values (localhost defaults). Actual secrets are in .env (gitignored) or Wrangler secrets. No real credentials ever in .env.example. |
| T-07-03 | Elevation of Privilege | WORKER_URL pointing to wrong endpoint if env var missing | mitigate | Both VITE_WORKER_URL and WORKER_URL are documented together in .env.example; README/docs can reference this single file. Fallback to localhost:8787 (dev default) means a missing var fails visibly in production (localhost not reachable) rather than silently routing to a wrong host. |
| T-07-04 | Tampering | wrangler.toml KV ID placeholder shipped to production | accept | Placeholder string is not a valid KV ID; Cloudflare Workers will reject the binding at deploy time, preventing accidental production deploy without replacement. The banner comment reinforces this. |
| T-07-05 | Denial of Service | CORS open to unintended origins after placeholder comment | mitigate | Origin list is explicit (two entries, no regex/wildcard); any origin not in the list gets a CORS error from the browser. Rust calls are unaffected. |
</threat_model>
