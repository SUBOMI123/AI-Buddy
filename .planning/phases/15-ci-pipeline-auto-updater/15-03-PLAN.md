<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installed app → latest.json endpoint | App fetches update manifest from GitHub Releases over HTTPS |
| latest.json → app binary (update install) | Update payload must have valid Ed25519 signature before install |
| CI → GitHub Releases (draft) | Draft release requires manual publish; no auto-publish to end users |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-03-01 | Tampering | Update payload without valid signature | mitigate | tauri-plugin-updater rejects any update where the Ed25519 signature in latest.json does not verify against the pubkey embedded in the binary; an attacker who compromises the GitHub Release page cannot ship a malicious update without also having the private key |
| T-15-03-02 | Spoofing | Fake latest.json served from CDN or MITM | mitigate | HTTPS enforced for all updater endpoint requests; GitHub's TLS cert pins the connection; signature verification provides a second layer independent of TLS |
| T-15-03-03 | Denial of Service | Updater check blocks app launch | mitigate | checkForAppUpdates() wraps all logic in try/catch — any network failure is silently swallowed; the call is fire-and-forget (not awaited) so it cannot block the SolidJS mount cycle |
| T-15-03-04 | Tampering | Version number in tauri.conf.json manipulated before tag | accept | tauri.conf.json is in source control; version is read at build time by tauri-action; any version manipulation would require a malicious commit — protected by repo access controls |
| T-15-03-05 | Information Disclosure | latest.json exposes release URLs | accept | Release URLs are already public on GitHub Releases; latest.json leaks nothing beyond what is already public; no credentials in the manifest |
</threat_model>
