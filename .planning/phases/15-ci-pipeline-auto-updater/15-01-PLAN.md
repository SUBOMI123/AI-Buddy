**Step 1 — Create src/updater.ts** with exactly this content:
```typescript
import { check } from '@tauri-apps/plugin-updater';
import { ask } from '@tauri-apps/plugin-dialog';
import { relaunch } from '@tauri-apps/plugin-process';

/**
 * Checks for app updates on launch. Silent if up to date.
 * Shows an in-app dialog if a newer version is available.
 * Called from SidebarShell onMount — fire-and-forget, errors swallowed.
 */
export async function checkForAppUpdates(): Promise<void> {
  try {
    const update = await check();
    if (!update) return; // No update available — silent

    const yes = await ask(
      `Version ${update.version} is available.\n\nInstall now?`,
      {
        title: 'Update Available',
        okLabel: 'Install & Restart',
        cancelLabel: 'Later',
      }
    );

    if (yes) {
      await update.downloadAndInstall();
      await relaunch();
    }
  } catch (e) {
    // Silently swallow network errors on launch — do not block startup
    console.warn('[updater] check failed:', e);
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| updater endpoint → app | App fetches latest.json from GitHub Releases; must verify signature before applying any update |
| GitHub Secrets → CI build | Private key and Apple credentials must never appear in CI logs |
| compiled binary → build artifacts | APP_HMAC_SECRET is embedded at compile time; must not appear in binary strings or debug output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01-01 | Tampering | tauri-plugin-updater / update payload | mitigate | Ed25519 signature verified by tauri-plugin-updater before applying any update; pubkey embedded in app binary — attacker cannot ship a valid update without the private key (ASVS L1: C-09) |
| T-15-01-02 | Information Disclosure | TAURI_SIGNING_PRIVATE_KEY in build logs | mitigate | Private key injected via `secrets.TAURI_SIGNING_PRIVATE_KEY` in workflow env — GitHub Actions masks secret values in all log output automatically; never echo or print the key |
| T-15-01-03 | Elevation of Privilege | Malicious update payload | mitigate | Ed25519 signature check in tauri-plugin-updater is mandatory and non-bypassable; any update with an invalid or missing signature is rejected before download completes |
| T-15-01-04 | Information Disclosure | APP_HMAC_SECRET in binary strings | accept | HMAC secret is embedded at compile time via env!() macro; binary strings analysis could expose it — accepted for beta (low-value target, no PII at stake); mitigate in v1.0 with server-side auth |
| T-15-01-05 | Spoofing | Malicious latest.json at update endpoint | mitigate | Endpoint is pinned to `github.com/SUBOMI123/AI-Buddy/releases` (not user-configurable); HTTPS enforced; signature verified before install — attacker would need to compromise the GitHub repo AND the private key |
</threat_model>
