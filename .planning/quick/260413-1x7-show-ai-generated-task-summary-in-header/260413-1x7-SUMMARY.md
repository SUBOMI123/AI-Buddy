# Phase quick Plan 260413-1x7: Show AI-generated task summary in header — Summary

## What Was Built

The overlay header previously echoed the raw user query verbatim ("Working on: how do I set up a scheduled dispatch?"). Now, after the AI response completes, the header shows a concise AI-generated phrase ("Set up a scheduled dispatch") extracted from a new `Task: ...` line Claude emits at the top of every step response.

**Changes:**

1. `parseSteps.ts` — new `ParseResult` interface `{ steps: Step[]; title: string }`. The `Task: ...` line is stripped before the `"1. "` compliance check and its phrase stored as `title`. `isClarifyingQuestion` signature unchanged.

2. `parseSteps.test.ts` — all 12 existing tests updated to destructure `{ steps }`. 3 new tests added: with Task line, without Task line, Task line with no subsequent steps. 15 total, all pass.

3. `ai.ts` — `SYSTEM_PROMPT` updated: `Task: {verb phrase}` is now required as the first line of every step response, followed immediately by `1.` on the next line. Clarifying question responses (single question, no steps) explicitly exclude the `Task:` line.

4. `SidebarShell.tsx` — `taskTitle` signal added. `onDone` destructures `parseSteps` result and calls `setTaskTitle(parsedTitle)`. Signal reset in `submitIntent` (start of new submission) and `handleNewTask`. Header span: `{taskTitle() ? taskTitle() : 'Working on: ${cleanLabel(lastIntent())}'}`.
