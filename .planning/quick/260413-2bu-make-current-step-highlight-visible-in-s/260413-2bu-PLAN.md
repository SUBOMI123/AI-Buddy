# Quick Task: Make current-step highlight visible in StepChecklist

## Problem

`StepChecklist.tsx` uses `var(--color-surface-secondary)` for the current step background.
In dark mode that resolves to `rgba(255, 255, 255, 0.08)` — 8% white on a dark surface —
which is imperceptible to users. The highlight that shows which step is active is invisible.

## Root Cause

The variable `--color-surface-secondary` was designed as a subtle surface tint, not a
selection/active-state indicator. Using it for an interactive highlight produces insufficient
contrast in dark mode.

## Fix

### 1. `src/styles/theme.css`

Add `--color-step-current` to both color scheme blocks:
- Light mode: `rgba(0, 122, 255, 0.10)` — light blue tint matching `--color-accent`
- Dark mode:  `rgba(10, 132, 255, 0.15)` — slightly stronger blue tint matching dark-mode accent

### 2. `src/components/StepChecklist.tsx`

Replace the background value on the current-step button:
```
// Before
background: isCurrent() ? "var(--color-surface-secondary)" : "transparent",

// After
background: isCurrent() ? "var(--color-step-current)" : "transparent",
```

## Files Changed

- `src/styles/theme.css`
- `src/components/StepChecklist.tsx`
