  <action>
    1. Add `createSignal` to the import from `solid-js` (already imports `For` and `Show`).

    2. Inside `StepChecklist` component body, declare a module-level signal:
       ```ts
       const [copiedIndex, setCopiedIndex] = createSignal<number | null>(null);
       ```
       Place it directly after `const currentStepIndex = ...`.

    3. In the copy button's `onClick` handler, after calling `copyToClipboard(...)`, set the feedback signal and schedule a clear:
       ```ts
       onClick={(e) => {
         e.stopPropagation();
         copyToClipboard(inlineCmd() ?? step.label);
         setCopiedIndex(index());
         setTimeout(() => setCopiedIndex(null), 1500);
       }}
       ```
       Note: `copyToClipboard` is async but `onClick` doesn't need to await it — the confirmation fires optimistically, matching the existing "silent fail" contract (D-09).

    4. Replace the static `<Clipboard size={14} />` inside the copy button with a conditional render:
       ```tsx
       <Show
         when={copiedIndex() === index()}
         fallback={<Clipboard size={14} />}
       >
         <span
           style={{
             "font-size": "11px",
             "font-weight": "600",
             color: "var(--color-accent)",
             "white-space": "nowrap",
           }}
         >
           Copied!
         </span>
       </Show>
       ```
       The `Check` icon is already imported but is semantically used for step completion — use the text label instead to keep the two affordances visually distinct.

    5. Update the copy button's `onMouseLeave` handler to restore color only when NOT in copied state:
       ```ts
       onMouseLeave={(e) => {
         if (copiedIndex() !== index()) {
           (e.currentTarget as HTMLButtonElement).style.color =
             "var(--color-text-secondary)";
         }
       }}
       ```
       This prevents the hover-out from overriding the accent color while "Copied!" is showing.

    No new imports are needed beyond `createSignal` (already in solid-js).
