**src/lib/parseSteps.ts — Add isClarifyingQuestion export:**

Export a new function directly below the existing parseSteps function:

```typescript
/**
 * Returns true when parseSteps produced exactly one step whose label ends with "?".
 * This heuristic detects clarifying question responses from the AI so the caller
 * can render prose (RawGuidanceText) instead of StepChecklist.
 *
 * Note: this function must only be called AFTER parseSteps — do not call on raw text.
 */
export function isClarifyingQuestion(steps: Step[]): boolean {
  return steps.length === 1 && steps[0].label.trimEnd().endsWith("?");
}
```

**Root cause of GAP-02:**
`SessionFeed` has its own `overflow-y: auto` and `flex: 1`, making it the scroll container.
`StepChecklist` is rendered as a sibling BELOW `SessionFeed` in the same flex column.
Since the checklist is outside `SessionFeed`'s scroll area, users cannot scroll up through
session history to reach content behind the checklist, and the checklist itself may be
clipped by the parent flex container.

**Fix in src/components/SessionFeed.tsx:**

Remove `overflow-y: auto` from the root div's style. Also change `flex: "1"` to just
`display: "flex"` and `flex-direction: "column"` without `flex: 1` — the outer `.sidebar-content`
in SidebarShell already has `flex: 1` and `overflow-y: auto`, so SessionFeed should be
a natural-height child that grows with its content. Specifically, change:

```
style={{
  flex: "1",
  "overflow-y": "auto",
  display: "flex",
  "flex-direction": "column",
  gap: "var(--space-sm)",
}}
```

to:

```
style={{
  display: "flex",
  "flex-direction": "column",
  gap: "var(--space-sm)",
}}
```

**Fix in src/components/SidebarShell.tsx — scroll ref target:**

The `sessionFeedRef` is currently set as the ref on `SessionFeed`. After removing the
internal scroll from SessionFeed, the auto-scroll target must shift to the outer
`.sidebar-content` div. Declare a new `let contentAreaRef: HTMLDivElement | undefined`
and attach it to the `.sidebar-content` div:

```tsx
<div
  class="sidebar-content"
  ref={(el) => { contentAreaRef = el; }}
  style={{
    flex: "1",
    "overflow-y": "auto",
    ...
  }}
>
```

Update the auto-scroll in `onToken` callback to use `contentAreaRef` instead of `sessionFeedRef`:

```typescript
if (contentAreaRef) {
  contentAreaRef.scrollTop = contentAreaRef.scrollHeight;
}
```

The `sessionFeedRef` ref callback on `SessionFeed` can be removed from props since it
no longer serves as a scroll target (the div it points to no longer scrolls). Remove
`ref={(el) => { sessionFeedRef = el; }}` from the `SessionFeed` JSX and remove the
`let sessionFeedRef` variable declaration.
