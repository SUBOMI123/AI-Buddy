## VOICE-02 Gap Detail

**Status:** Partial — wired but not ergonomic

**What works:**
- Per-line play buttons in `GuidanceList.tsx` call `playTts(line)` — functional TTS available on any guidance line
- Full pipeline: `cmd_play_tts` → Worker `/tts` → ElevenLabs → rodio playback
- Auto-play code exists in `SidebarShell.tsx` `onDone` callback (gated on `ttsEnabled()`)

**What's missing:**
- In-app toggle to enable `ttsEnabled` — only `setTtsEnabled(false)` is called on load (default off)
- Settings screen has no voice preferences section

**Impact:** Users can use TTS via per-line play buttons. Auto-play (hands-free experience) is inaccessible without editing `settings.json` directly. Not a blocker — per-line TTS satisfies "voice output via TTS" — but reduces UX quality.
